## Risk Mitigation

| Risk | Mitigation |
|------|------------|
| Binary build fails for platform | Use `continue-on-error` + manual build |
| GitHub Action rate limits | Cache binaries aggressively |
| Install script fails | Provide manual download fallback |
| Backward compatibility break | Run v1.0 policy test suite before release |

---

## Success Criteria

v1.1.0 is successful when:

1. **15-minute adoption**: New user can run `assay coverage` in CI within 15 minutes
2. **Zero-friction install**: `curl | sh` works on Linux/macOS
3. **GitHub Action works**: PR check completes in <30 seconds
4. **Design partner validation**: Parloa or Celonis confirms CI integration working

---

## Post-Release (v1.2.0 Planning)

After v1.1.0 ships, prioritize:

1. **Python SDK** - `pip install assay` with PyO3 bindings
2. **Explain graduation** - Move from experimental to stable
3. **Baseline management** - "main = baseline" for regression detection
4. **LangChain adapter** - Import traces from LangChain runs
